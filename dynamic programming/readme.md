# 动态规划(Dynamic Programming)导读
DP发明人Richard Bellman（同时也是Bellman-Ford算法里的Bellman）在发明DP时正在为美国空军工作，为了要经费，他挑选了 “Dynamic”（动态）和 “Programming”（规划/编程）这两个词

实际上，DP既不动态，也不规划，实际上是一系列的优化组成

0. 暴力搜索
1. 记忆化搜索（DP中的“状态”出现了）
2. 数学式子描述状态之间的关系（DP中的“状态转移方程”出现了）
3. 递归改成循环，改变了遍历信息的顺序；同时使得信息可以以更密集的方式存储，也就使得“滚动数组”成为了可能
4. 数学上的等价变换，优化时间复杂度
    * 决策点剔除（凸包优化，斜率优化，四边形不等式优化，单调队列优化等）
    * 矩阵加速（线性转移看成矩阵乘法，矩阵快速幂加速转移）
5. 和图论的联系：
    * 状态看成点，转移看成边
    * 无后效性: 图中不存在环，即DP必须在有向无环图上进行
    * 最优子结构：“A到B到C”是“A到C”的最优路径，那么这段最优路径中截取的“B到C”路径，一定是“B到C”的最优路径
    * 最值问题可以看成最短路/最长路，可否问题可以看成连通性问题

## 一些推状态的技巧
数据小有可能是DP，将输入数据直接作为状态是典题

增量式DP：以XX为结尾的串/序列，后面加一个XX

如果DP取值是1或0，不妨把一维作为存储的数值去处理，而不是维度

整除只在余数为0时变换，可以用同余聚类

连续m个元素至少选2个，考虑最后两个元素选择在什么位置，可以把倒数第二个元素绝对位置，空间复杂度O(n)，优化成倒数第二个元素和倒数第一个元素的相对位置，空间复杂度O(m)。同时倒数第一个元素的位置考虑对m取模，这样第一维也优化成O(m)

对于dp[i][j]从max(dp[i-1][j-1],dp[i-2][j-2],dp[i-3][j-3]...)转移过来的，这样对角线上的DP，每条对角线用i-j作为标识符，求DP时同时处理每条对角线前缀最大值

设计状态时尽量利用已知信息，[例子](https://www.luogu.com.cn/problem/P1387)设计的好可以 $O(nm)$，[类似的例子](https://codeforces.com/problemset/problem/1393/D)，这两个例子都是按照一定顺序遍历，要找的图形中，最后一个被遍历的格子作为设计的状态，最大程度利用了已知信息
## 一些推转移的技巧
合并同类项，比如 $a_i-a_j (中间是不等式符号) i-j$ 把含有 i 的移项到一边，含有 j 的移项到另一边，即 $a_i-i (中间是不等式符号) a_j-j$

对于更新A，A不能更新B；更新B，B不能更新A，的情况，需要拿更早的状态更复杂的转移来解决
## 一些优化的技巧
如果转移式子只和上一层有关，滚动数组

观察转移式子中各项的单调性，是否能推导出最优决策点的单调性

遍历一棵树，设当前遍历到 $u$ 节点，有子节点 $v_1,v_2,...$，每次遍历一颗子树就把信息和前几颗子树的信息合并，放到 $u$ 上，最终总的复杂度是 $O(n^2)$。因为每对不同点对 $(x,y)$ 不会重复合并，最多 $O(n^2)$ 个不同点对

一颗树的高度小于等于树大小，所以合并复杂度如果和高度相关，不会超过 $O(n^2)$

记忆化搜索DP时，后续状态都搜索过了，但是前继状态不一定，这种情况倒着DP会更好

DP优化可以考虑调换内外循环顺序

如果双层循环中，每轮内层循环的转移视为一个集合，在外层循环迭代前称为集合A，迭代后称为集合B，如果A是B的子集，可以考虑外层循环中只维护B对A的补集，这样能省掉内层循环，形如下列代码
```cpp
for(int i=0;i<n;++i)
    for(int j=0;j<i;++i)
        d[i]=min(d[i],d[j]);
```

不要实时计算，就要提前计算并存储
