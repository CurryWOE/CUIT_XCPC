# 并查集
## 阿克曼函数与反阿克曼函数
阿克曼函数增长很快，下为定义

$A(m,n)=\begin{cases}n + 1 & m = 0 \\ A(m-1, 1) & m > 0 \text{ and } n = 0 \\ A(m-1, A(m, n-1)) & otherwise \end{cases}$

由于阿克曼的反函数无法用初等函数描述，科学家使用“反阿克曼函数”作为阿克曼函数的广义逆，不是阿克曼函数的反函数。定义如下

$\alpha(m,n) = \min \left\{ k \ge 1 \bigm| A\left(k, \left\lfloor \frac{m}{n} \right\rfloor \right) > \log_2 n \right\}$

当m和n同阶，反阿克曼函数的单参数定义如下

$\alpha(n) = \min \left\{ k \ge 1 \bigm| A\left(k, 1 \right) > \log_2 n \right\}$

| α(n) 的值 | n 的范围 |
|:---|:---|
| 1 | n < 4 |
| 2 | $4 \le n < 16$ |
| 3 | $16 \le n < 256$ |
| 4 | $256 \le n < 65536$ |
| 5 | $65536 \le n < 2^{65536}$ |
| j | $2^{2^{j-1}} \le n < 2^{2^j}$ |
## 复杂度
以下为路径压缩时，各种情况的时间复杂度，表格摘自Tarjan论文，n是元素数，m是操作数

||朴素合并|按秩合并或者启发式合并|
|---|---|---|
|$m\ge n$|$m\log_{1+m/n}n$|$m\alpha(m,n)$|
|$m<n$|$n+m\log n$|$n+m\alpha(n,n)$|

一般来说路径压缩+朴素合并就够用了，计算复杂度使用5是比较简便的做法
```cpp
int f[N],rk[N];
void init(int n)
{
    memset(f,-1,sizeof(int)*(n+1));
    memset(rk,0,sizeof(int)*(n+1));
}
int find(int x)
{
    return f[x] == -1 ? x : (f[x] = find(f[x]));
}
void merge(int u, int v)
{
    int x = find(u), y = find(v);
    if(x==y)
        return;
    if (rk[x] < rk[y])
        swap(x,y);
    f[y]=x;
    if (rk[x] == rk[y])
        ++rk[x];
}
```

# 带权并查集
在路径压缩和并集的时候，维护点和根之间的信息

已知(0,4],(2,4]的区间和，问给出的(0,2]区间和是否等于(0,4]区间和减(2,4]区间和

每次给出的区间若不在一个连通块，则并集，区间和作为非根点的点权；否则检查区间和是否合法（前缀和）
